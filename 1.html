2. 签名验证机制

  • 筛选参数

所有请求包含的非空值参数，sign字段不参与签名

  • 排序

将筛选的参数按照第一个字符的键值ASCII码递增排序（字母升序排序），如果遇到相同字符则按照第二个字符的键值ASCII码递增排序，以此类推。

  • 拼接参数

将排序后的参数与其对应值，组合成 参数=参数值 的格式，并且把这些参数用 & 字符连接起来，值为空的参数需要需要设置默认值，数字类型设置为0，字符串类型设置为 空字符串""，禁止出现Null字样

  • 签名生成sign

签名采用 MD5 32位加密方式，上述拼接完成的字符串经过MD5 32位加密并转成大写字符得到加密结果。得到最终的sign

示例:

商户UUID：eebdf103-39c6-4ce6-998c-95a32bce7ab3

密钥：fMoxMzCcVjHUqMD32gGeYPayzmuOGyxq

未加sign签名字段的请求参数json：

{
    "merchantUUID":"eebdf103-39c6-4ce6-998c-95a32bce7ab3",
    "price":"199.30",
    "payType":"alipay_h5",
    "merchantOrderNo":"211005023628200453",
    "notifyUrl":"http://127.0.0.1/api/gateway/notifyInnerMerchantOrder",
    "returnUrl":"http://127.0.0.1/pay/jsp/gopay/true.jsp",
    "remark":"下单请求"
}

排序并拼接上秘钥后的待加密字符串:

merchantOrderNo=211005023628200453&merchantUUID=eebdf103-39c6-4ce6-998c-95a32bce7ab3¬ifyUrl=http://127.0.0.1/api/gateway/notifyInnerMerchantOrder&payType=alipay_h5&price=199.30&remark=下单请求&returnUrl=http://127.0.0.1/pay/jsp/gopay/true.jsp&key=fMoxMzCcVjHUqMD32gGeYPayzmuOGyxq

签名结果sign:(32位 MD5后转大写)

91381A854EC9AD02D409B1110955A5BE

最终提交到下单接口的json内容:

3. 统一下单（接口）

a. 请求说明

  • 路由地址：/api/gateway/generateOrder
  • 请求方式：POST
  • 接收参数类型(Content-Type): application/json

b. 请求参数

参数名 类型 必填 说明
merchantUUID string 必填 商户UUID，从后台右上角获取
price string 必填 订单金额，最大支持精确到小数点后2位,例：199.30
payType string 必填 支付方式（通道编码），详情见附录1:通道编码payType列表
clientIp string 必填 玩家真实IP，如果有传值，参与签名
merchantOrderNo string 必填 商户订单号
notifyUrl string 必填 支付结果异步回调地址
returnUrl string 必填 支付返回重定向地址
remark string 非必填 订单备注，转卡通道请在此处填入会员真实姓名用来核实到账
sign string 必填 参数签名，规则见前文，签名规则

请求参数示例:

{
    "merchantUUID":"eebdf103-39c6-4ce6-998c-95a32bce7ab3",
    "price":"199.30",
    "payType":"alipay_h5",
    "merchantOrderNo":"211005023628200453",
    "notifyUrl":"http://127.0.0.1/api/gateway/notifyInnerMerchantOrder",
    "returnUrl":"http://127.0.0.1/pay/jsp/gopay/true.jsp",
    "remark":"下单请求",
    "sign":"91381A854EC9AD02D409B1110955A5BE"
}

b. 返回结果:data内部参数，只有当外层code=0的时候才会有值，code为非0的时候，msg中有提示失败原因

参数名 类型 说明
sysOrderNo string 系统订单号
merchantOrderNo string 商户提交的订单号
payUrl string 支付链接
payParams string 支付参数，一般情况下没有值，只有个别支付入金网关例如支付宝sdk支付的时候需要
price string 订单金额
expiresTime string 订单过期时间

返回结果示例

{
    "code": 0,
    "data": {
        "sysOrderNo": "JD88882205310013030001",
        "merchantOrderNo": "211005023628200453",
        "payUrl": "http://xx.zhuankehaoyangmao.com:7788/dsx/pay.do?method=codePrePay&token=ukedhciephdywjdJD88882205310008130001acfsv",
        "payParams": "",

  • 请求参数类型(Content-Type): application/json

b. 请求参数

参数名 类型 说明

status string 订单状态,"PAY_STATUS_SUCCESS" 为支付成功
fee string 订单产生的手续费,double类型
payTime string 支付时间
realMoney string 实际支付金额，根据不同通道规则，跟订单发起金额对比，最大可能会有上下1元的浮动
sign string 参数签名，规则见前文，签名规则

    "sysOrderNo":"JD88882205310035500001",

    "fee":"19.93",
    "payTime":"2022-05-31 00:36:04",
    "realMoney":"199.3",
    "sign":"5F9A02235A526925AE6D2CD62FCACE8D"
}

c. 回调接收返回

我方调用notifyUrl回调你方系统之后，你方系统收到回调并且再次调用我方查单接口核实订单状态，以确认是我方系统发送过来的回调，防止黑客伪造回调发送假回调导致损失，如未对接反查订单状态接口，导致被撸单，我方概不负责，核实处理完成后需要给我方回复SUCCESS,才会确认该笔订单回调成功

我方接收返回结果（必须为大写）

SUCCESS

3. API代付下单（接口）

a. 请求说明

  • 路由地址：/api/gateway/walletWithdrawAPIV2
  • 请求方式：POST
  • 接收参数类型(Content-Type): application/json

b. 请求参数

参数名 类型 必传参数 参与签名 说明
merchantUUID string 必传 是 商户UUID，从后台右上角获取
amount string 必传 是 订单金额，将浮点数保留两位小数后转换成string类型参与签名和传输
channelCode string 必传 是 通道编码（eg：301）
productType string 必传 是 代付目标类型，用来区分选择银行卡代付还是支付宝代付（银行卡代付：chn.bank.payout 支付宝代付：chn.alipay.payout）
currency string 必传 是 货币代码，eg：CNY （人民币）
approveType string 必传 是 选择提现审核方式, 1 (人工审核打款) 2 (API三方代付) 此处目前默认传2
merchantOrderNo string 必传 是 商户订单号
notifyUrl string 必传 是 支付结果异步回调地址
remark string 选填 是 订单备注(选填)
extInfo object 必传 否 代付附加信息，具体内容包含在json对象里边！！！该字段内容不参与签名
extInfo.bankName string 必传 否 银行名称：eg:中国银行 （支付宝代付填写固定值：支付宝）
extInfo.fullName string 必传 否 持卡人姓名：eg:张三 （支付宝代付也是一样，填写支付宝用户实名）
extInfo.bankcardNumber string 必传 否 持卡人卡号：eg: 6217876621231133 (支付宝代付填写支付宝帐号eg:19872333321)
sign string 必传 否 参数签名结果，规则见前文，MD5签名规则，该字段不参与签名

请求参数示例:

{
    "merchantUUID":"eebdf103-39c6-4ce6-998c-95a32bce7ab3",
    "amount":"100.00",
    "channelCode":"301",
    "productType":"chn.bank.payout",
    "currency":"CNY",
    "approveType":"2",
    "remark":"just pay",
    "merchantOrderNo":"96270A93765f113xx8",
    "notifyUrl":"http://localhost/api/gateway/notifyInnerMerchantWithdrawOrder",
    "extInfo":{
        "bankName":"中国银行",
        "fullName":"张三",
        "bankcardNumber":"6217876621231133"
    },
    "sign":"GTYmRuqm80OVNaDXMEUz2aUqgNJ2411"
}

b. 返回结果

参数名 类型 说明
sysOrderNo string 系统订单号
merchantOrderNo string 商户提交的订单号
amount string 代付提交金额
fee string 代付提现手续费
realMoney string 实际代付打款
status int 代付提现订单状态 3，代付处理中，6，代付提交失败

返回结果示例

{
    "code": 0,
    "data": {
        "sysOrderNo": "WD2108241401560002",
        "merchantOrderNo": "IME12134324123123123",
        "amount": "100",
        "fee": "10",
        "realMoney": "100",
        "status": 3
    },
    "msg": "操作成功"
}
